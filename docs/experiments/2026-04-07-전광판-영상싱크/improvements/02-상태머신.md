# 재로딩만 → 3단계 상태 전환

> R1-R4에 걸쳐 "무조건 다시 로딩" → "어긋남 정도에 따라 다르게 대응"

## 문제

현재 시스템은 오차가 허용치를 넘으면 **무조건 영상을 다시 로딩**함. 이러면 아주 조금 어긋난 것(50~100ms)도 영상이 끊기면서 다시 시작됨. 사람 눈에 깜빡임으로 보임.

## 단독 설계였다면

"어긋나면 다시 로딩하면 되지" — 끊김을 줄이려는 시도 자체가 없었을 것.

## 적대적 검증에서 일어난 일

**R1** — Executor가 "약간 어긋남은 속도 조정, 많이 어긋남은 재로딩"으로 2단계 제안
**R1-C5** — Challenger: "속도 조정 시작 기준(200ms)과 떨림 방지 기준(100ms)이 겹쳐서 같은 오차에 다른 동작이 가능하다"
**R4** — 상태 전환 방식으로 재설계하여 기준 충돌 해소

## 최종 설계

```
[정상] 오차 50ms 미만 → 아무것도 안 함
   │
   │ 오차가 100ms를 넘으면
   ↓
[약간 어긋남] 50~200ms → 재생 속도를 살짝 빠르게/느리게 (0.98~1.02배)
   │
   │ 오차가 200ms를 넘으면
   ↓
[크게 어긋남] 200ms 초과 → 영상 다시 로딩
```

**떨림 방지**: "약간 어긋남"으로 들어가는 기준은 100ms, "정상"으로 돌아오는 기준은 30ms로 다르게 설정. 이렇게 하면 경계에서 왔다 갔다 하는 떨림을 막을 수 있음.

## 뭐가 좋아졌나

| 항목 | 이전 | 이후 |
|------|------|------|
| 약간 어긋남 | 재로딩 → 화면 끊김 | 속도 조정 → **끊김 없음** |
| 떨림 방지 | 없음 | 들어가는 기준 ≠ 나오는 기준 |
| 상태 추적 | 없음 | 지금 상태가 뭔지 로그로 확인 가능 |

## 아직 해결 안 된 것

- 영상 재생기가 속도 조정을 지원 안 하면? → 대안 동작 미정의 (R5-C1)
- 100ms 기준과 50ms 기준 사이(50~100ms)의 상태가 약간 모호 (R4-C3, 약한 지적)
