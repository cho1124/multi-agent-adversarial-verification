# 재로딩 전용 → 3단계 상태머신

> R1-R4에 걸쳐 단일 보정 방식 → 오차 크기별 분기 제어

## 문제

현재 시스템은 오차가 허용치를 넘으면 **무조건 영상 재로딩**. 소오차(50~200ms)에도 재로딩하면 화면 끊김이 발생하여 체감 품질 저하.

## 적대적 검증에서 일어난 일

**R1** — Executor가 2단계 보정(속도보정 + 재로딩) 제안
**R1-C5** — Challenger: "히스테리시스(100ms)와 소오차보정(<200ms)의 우선순위가 충돌"
**R4** — Executor가 상태 머신으로 재정의하여 우선순위 충돌 해소

## 최종 설계

```
SYNCED (<50ms) ─── 오차 증가 (>100ms) ───→ DRIFTING (50~200ms, 속도보정)
    ↑                                           │
    └── 오차 감소 (<30ms) ──────────────┘       │
                                                  │ 오차 증가 (>200ms)
                                                  ↓
                                             DESYNCED (재로딩)
```

히스테리시스가 **상태 전이 조건에만** 적용되어 정책 충돌 제거.

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 보정 방식 | 재로딩만 | 3단계 (유지/속도보정/재로딩) |
| 소오차 처리 | 재로딩 (끊김) | 속도 미세조정 (끊김 없음) |
| 진동 방지 | 없음 | 히스테리시스 (적용≠해제 임계값) |
| 상태 추적 | 없음 | 명시적 상태 + 전이 이력 로깅 |

## 미해소

- 속도보정 미지원 디코더에서 DRIFTING 동작 폴백 미정의 (R5-C1)
- 50~100ms 구간의 히스테리시스 해석 모호 (R4-C3, WEAK)
