# 수동 조정 → 자동화

> R1-R3에 걸쳐 "사람이 값을 찍어 넣는 방식"의 한계 해소

## 문제

현재 시스템은 딜레이(290ms)와 허용치(80ms)를 **스케줄 파일에 사람이 직접 입력**. 매뉴얼에도 "많은 시도를 통해 통계적으로 적당한 값을 찾는 것이 중요합니다"라고 적혀 있음.

문제점:
- 사람마다 다른 값을 넣을 수 있음 (재현성 없음)
- 콘텐츠가 바뀌면 값도 바뀌어야 하는데 모를 수 있음
- 숫자를 잘못 입력하면 더 나빠질 수 있음
- 여러 PC에 같은 값을 일일이 넣어야 함

## 단독 설계였다면

"매뉴얼 보고 조정하세요" — 지금이 이 상태.

## 적대적 검증에서 일어난 일

Challenger 독립 체크리스트 **CL15**: "수동 파라미터 운영은 재현성과 신뢰성을 떨어뜨리므로, 문제 원인이 설계인지 운영인지 분리해야 한다"

이 지적을 시작으로 3개 개선이 연쇄적으로 나옴:
1. 딜레이 → 5단계 자동 측정으로 대체 (사람이 290을 넣을 필요 없음)
2. 허용치 → 상태 전환 방식으로 대체 (사람이 80을 넣을 필요 없음)
3. 모니터링 → 자동 로그 + 분포 분석으로 대체 (눈으로 확인할 필요 없음)

## 뭐가 좋아졌나

| 항목 | 이전 | 이후 |
|------|------|------|
| 딜레이 값 | 사람이 시행착오로 결정 | 시스템이 자동 측정 |
| 허용치 | 사람이 시행착오로 결정 | 상태 전환이 자동 관리 |
| 문제 진단 | 서버 화면 보고 빨간 글씨 확인 | 오차 분포 로그 + 통계 분석 |
| 콘텐츠 변경 시 | 사람이 다시 값 조정 | PC별/콘텐츠별 자동 프로파일링 |
| 여러 PC 배포 | 스케줄 파일 일일이 복사 | 서버에서 중앙 관리 |
| 재현성 | 없음 (사람 의존) | 측정 데이터 기반으로 언제든 재현 |
