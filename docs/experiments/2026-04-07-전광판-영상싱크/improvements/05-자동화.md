# 수동 조정 → 자동화

> R1-R3에 걸쳐 스케줄 파일 수동 조정의 한계 해소

## 문제

현재 시스템은 딜레이(290ms)와 허용치(80ms)를 **스케줄 파일에 수동 입력**. "많은 시도를 통해 통계적으로 적당한 값을 찾는 것이 중요"하다고 매뉴얼에 명시. 재현성 없고 운영 부담이 큼.

## 적대적 검증에서 일어난 일

Challenger 독립 체크리스트 CL15에서 "수동 파라미터 운영은 재현성과 신뢰성을 떨어뜨리므로 문제 원인이 설계인지 운영인지 분리해야 한다"고 사전 지적.

## 최종 설계

| 항목 | 수동 | 자동 |
|------|------|------|
| 딜레이 | 스케줄 파일 수동 입력 | 단계별 계측 기반 자동 추정 |
| 허용치 | 스케줄 파일 수동 입력 | 상태머신 히스테리시스 자동 전이 |
| 문제 진단 | 서버 Agent 화면 육안 확인 | 오차 분포 로그 + 자기상관 분석 |
| 콘텐츠별 조정 | 불가 | 단말/콘텐츠별 프로파일링 |

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 파라미터 조정 | 사람이 시행착오 | **시스템이 자동 적응** |
| 재현성 | 없음 (사람 의존) | 계측 데이터 기반 재현 가능 |
| 배포 일관성 | 스케줄 파일 수동 복사 | 서버 중앙 프로파일 관리 |
| 오입력 위험 | 있음 | 제거 (수동 입력 불필요) |
