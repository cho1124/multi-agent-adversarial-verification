# 50ms이하 오차 원인 분리

> R2-R4에 걸쳐 "허용치를 줄이면 진동한다"의 근본 원인 식별

## 문제

현재 시스템에서 허용치를 50ms 이하로 줄이면 **연속적으로 동기가 맞지 않는 경향**이 있음. 이것이 실제 드리프트인지, 측정 노이즈에 대한 과민 반응인지 구분이 안 됨.

## 적대적 검증에서 일어난 일

**R2-C6** — Challenger: "히스테리시스는 제어 정책일 뿐 원인 분리가 아니다"

→ Executor가 자기상관(autocorrelation) 분석 도입

## 최종 설계

SYNCED 상태에서 최근 60초(60샘플)의 오차 시계열 수집:

- **자기상관 ≈ 0 (무상관)** → **노이즈 우세**: 허용치+히스테리시스로 충분. 속도보정 불필요.
- **자기상관 > 0 (양의 상관)** → **드리프트 우세**: 미세 속도보정을 상시 적용하여 드리프트 추종.

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 50ms이하 대응 | 허용치 올리기 (포기) | **원인별 분기 대응** |
| 노이즈 vs 드리프트 | 구분 불가 | 자기상관으로 실시간 판별 |
| 보정 전략 | 일률적 | 노이즈: 무시, 드리프트: 속도보정 |
