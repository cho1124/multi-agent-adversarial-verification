# TCP → UDP 멀티캐스트

> R1-R5에 걸쳐 실시간 동기 신호의 전송 프로토콜 전환

## 문제

TCP의 재전송, Nagle 알고리즘, 헤드오브라인 블로킹이 동기 신호의 **지연 편차(지터)**를 증가시켜 보정 정확도를 떨어뜨림.

## 적대적 검증에서 일어난 일

**R1-C3** — Challenger: "UDP 전환 시 손실/중복/역순 도착 복구 규칙이 없다"
**R2-C2** — Challenger: "신호 누락 시 마지막 상태 유지가 드리프트를 억제하지 못한다"

→ R5에서 마지막 상태 유지를 폐기하고 자체 드리프트 보상 모드로 전환

## 최종 설계

- UDP 멀티캐스트 (내부망, 브로드캐스트 대비 효율적)
- 메시지 포맷: `{seq_no, server_qpc, media_id, media_hash, playback_position}`
- 3연속 누락 → 클라이언트 자율 드리프트 보상 (최근 드리프트율 기반)
- 신호 복구 시 즉시 서버 기준 재동기

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 프로토콜 | TCP (재전송 지연) | UDP 멀티캐스트 (최소 지연) |
| 지터 | TCP 메커니즘에 의존 | 프로토콜 레벨 지터 최소화 |
| 신호 누락 | TCP가 보장 | 시퀀스 번호 + 자율 보상 |
| 확장성 | 클라이언트별 개별 연결 | 멀티캐스트 1회 전송 |
