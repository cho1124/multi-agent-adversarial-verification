# 전광판 영상 동기화 시스템 적대적 검증 실험 (2026-04-07)

> 내부망 TCP/IP 기반 전광판 영상 플레이어 동기화 시스템의 현재 문제 분석 + 개선안 설계

---

## 결론

### 실무 결론: "핵심 결함은 네트워크가 아니라 시간 기준 설계"

| 지표 | 현재 시스템 | 적대적 검증 후 설계 |
|------|-----------|-------------------|
| 시간 기준 | 미정의 (서버 재생 시각만 전송) | QPC 모노토닉 + NTP 교정 + **교차 단말 매핑 필요** |
| 보정 방식 | 재로딩만 (끊김 발생) | 3단계 상태머신 (SYNCED/DRIFTING/DESYNCED) |
| 딜레이 보상 | 고정 290ms (수동 조정) | 단계별 계측 기반 적응형 (폐기됨: EMA 단일 지표) |
| 허용치 | 고정 80ms (수동 조정) | 히스테리시스 기반 상태 전이 |
| 프로토콜 | TCP | UDP 멀티캐스트 + 시퀀스 번호 |
| 50ms이하 불일치 | 원인 불명 | **노이즈 vs 드리프트 자기상관 분석** |
| 관측성 | 서버 Agent 로그만 | 단계별 지연 계측 + 오차 분포 + 외부 계측 |

### 미해소 Critical 모순 (사람 판단 필요)

| ID | 모순 | 영향 |
|----|------|------|
| **R4-C1/R7-C1** | QPC는 단말별 로컬 클럭 — 서버 QPC와 클라이언트 QPC를 직접 비교할 수 없음. 교차 단말 매핑 규칙 미정의 | **전체 오차 계산식의 근간이 무너짐** |
| **R5-C1** | 속도보정 미지원 디코더에서 DRIFTING 상태 동작 미정의 | 상태머신 부분 붕괴 |
| **R5-C2/R7-C4** | 비루프 콘텐츠 전환 규칙 없음 | 비루프 콘텐츠 미지원 |
| **R5-C3** | 즉시 재생 + 재로딩 = 재생공백 0 모순 | 복구 전략 내부 충돌 |
| **R7-C3** | LOOP_START 3회 전부 손실 시 처리 없음 | 루프 경계 취약 |

### 검증된 것

1. **독립 체크리스트가 핵심 누락을 사전 포착** — 25항목 중 CL2(기준 클럭)가 7라운드 내내 미해소로 남아 가장 근본적인 문제임을 드러냄
2. **프레임 전환 감지 작동** — R2-C1(PTP로 제약조건 교체), R3-C4(대기를 정책으로 재정의) 감지
3. **Executor Collapse 감지** — R4부터 Arbiter가 Executor 설계의 붕괴를 판정. 대칭적 검증의 효과
4. **Arbiter VALID 비율** — R1-3: 24/28건 VALID(86%), R4-7: 10/14건 VALID(71%)

### 검증되지 않은 것 (한계)

1. QPC 교차 단말 매핑의 기술적 해소안이 도출되지 않음 — 추가 연구 필요
2. Windows 환경 전용 — 다른 OS에서의 적용 미검증
3. 실제 전광판 하드웨어에서의 테스트 미실시

## 최종 확정 설계 (부분 — 미해소 모순 존재)

### 확정된 부분

**제어 상태머신:**
```
SYNCED (<50ms) ─── 오차 증가 ───→ DRIFTING (50~200ms)
    ↑                                    │
    └── 오차 감소(30ms 이하) ──────┘     │
                                          │ 오차 증가 (>200ms)
                                          ↓
                                     DESYNCED (재로딩)
```

**단계별 계측:**
```
동기 신호 수신 → [network_delay] → 오차 계산
재로딩 시작 → [decode_init] → [buffer_fill] → [first_frame] → [display_delay] → 재생
```

**50ms이하 진단:** 자기상관 분석 — 노이즈(무상관) vs 드리프트(양의 상관) 구분

**루프 동기:** LOOP_START 3회 반복 전송 + 강제 재동기

**콘텐츠 정합성:** media_id + media_hash + 백그라운드 다운로드

### 미확정 (미해소 모순)

- **시간 기준**: QPC + NTP 조합의 교차 단말 매핑 규칙
- **속도보정 미지원**: 상태머신 DRIFTING 동작의 폴백
- **비루프 콘텐츠**: 전환 규칙
- **복구**: 재생공백 0 보장 방법
- **외부 계측**: QPC 대신 NTP 교정 시각 기반 타임코드로 전환 필요

## 핵심 전환점

### R4 — QPC 교차 단말 매핑 문제

Executor가 모든 시각을 QPC(QueryPerformanceCounter) 기반으로 통일하겠다고 제안했으나, Challenger가 "QPC는 단말별 로컬 모노토닉 클럭이라 서버 QPC와 클라이언트 QPC를 직접 비교할 수 없다"고 지적. 이 모순이 R7까지 해소되지 않아 **Arbiter가 Executor Collapse를 판정**.

이 발견은 동기화 시스템 설계에서 **가장 근본적인 문제** — "두 시계를 어떻게 같은 축에 놓는가"가 해소되지 않으면 나머지 모든 보정 로직이 무의미해진다는 것을 보여줌.

### R2 — 프레임 전환 2회 감지

1. Executor가 NTP 한계를 인정하면서 PTP를 대안으로 제시 → Arbiter가 "원인 특정 없이 제약조건 교체"로 판정
2. Executor가 워치독 재시작 복구를 "대기"로 처리 → Arbiter가 "서비스 연속성을 정책으로 재정의"로 판정
