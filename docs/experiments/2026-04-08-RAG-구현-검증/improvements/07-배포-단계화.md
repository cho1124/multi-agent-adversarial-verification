# 배포 단계화

> 소주제2 (비용 제로 배포) — R3~R4에서 확정

## 문제

"다른 사람들이 사용할 수 있도록"의 범위가 불명확. 개발자가 clone하는 것과 비개발자가 웹에서 쓰는 것은 완전히 다른 요구사항.

## 적대적 검증에서 일어난 일

**R3 — Challenger 지적:**

"현재 진입점이 `python scripts/chat.py` (CLI). 비개발자 접근 불가. 무료 호스팅(Vercel/Railway/HuggingFace Spaces)에서 Ollama를 돌릴 수 있는가?"

**R4 — 수렴:**

"링크 클릭만으로 사용"은 상시 가용 서비스를 전제하는데, 무료 GPU tier는 세션 2시간 제한. 비개발자 대상은 장기 목표로 분리.

## 확정된 3단계 로드맵

### Phase 1: 개발자 대상 (현재 목표)

```
형태: GitHub repo + pip install + CLI
진입: git clone → pip install -r requirements.txt → python scripts/chat.py
선결: 소주제1 CRITICAL 버그 수정 (벡터 차원 통일)
목표: GitHub Star + Fork + 피드백 수집
```

**최소 요구사항:**
- README에 설치 3줄 명시
- Ollama 사전 설치 안내
- `.env.example` 제공 (유료 API 키는 optional)

### Phase 2: 준개발자 대상

```
형태: Docker Compose + localhost 웹 UI
진입: docker compose up → localhost:8000 접속
선결: citation 준수 실험 완료 + Gradio/Streamlit UI
목표: "docker compose up" 한 줄로 완전 실행
```

**Docker Compose 구성:**
```yaml
services:
  ollama:
    image: ollama/ollama
    # 모델 자동 pull
  app:
    build: .
    ports: ["8000:8000"]
    depends_on: [ollama]
```

### Phase 3: 비개발자 대상 (장기)

```
형태: HuggingFace Spaces 또는 호스팅
진입: URL 클릭
선결: 비용 모델 결정 (GPU 무료 tier 2시간 제한 문제)
목표: 링크 공유만으로 사용 가능
```

**현실적 선택지:**
- HF Spaces + CPU: 느리지만 상시 가용 (gemma2:2b)
- HF Spaces + GPU: 빠르지만 2시간 제한 (gemma2:9b)
- 자비 호스팅: 소규모 VPS + Ollama ($5/월 수준)

## 핵심 원칙

> Phase 1을 완벽하게 → Phase 2 → Phase 3 순서. 동시 진행 금지.

3-tier 유지보수 폭발(R2에서 기각)의 교훈: 하나를 완벽하게 만든 뒤 확장.
