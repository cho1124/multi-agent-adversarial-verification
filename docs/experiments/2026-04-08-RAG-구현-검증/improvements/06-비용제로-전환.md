# 비용 제로 전환

> 소주제2 (비용 제로 배포) — R1~R4에 걸쳐 확정

## 문제

현재 시스템이 OpenAI(임베딩+LLM) + Anthropic(LLM) + Supabase(DB) 유료 API에 의존하여, 다른 사람이 사용하려면 **월 $20~45 비용** 발생.

## 적대적 검증에서 일어난 일

### R1 — "전부 로컬로 바꾸면 된다" 기각

Executor 초기안: OpenAI→gte-small, LLM→Ollama, DB→ChromaDB

Challenger 4건 VALID 반박:
1. **임베딩 품질 전파**: MTEB retrieval 62.3→57.8 (4.5점 하락) + 한국어 도메인 미검증
2. **로컬 LLM 지시 준수**: 8B 모델의 citation `[1]` 형식 준수율 미보장
3. **CPU 추론 속도**: 2-5 tok/s → 답변 60-150초 대기 (서비스 불가)
4. **ChromaDB hybrid**: BM25 없음 → hybrid search 불가

### R2 — 3-tier 구조도 기각

Executor 수정안: Tier 1(로컬) / Tier 2(무료API) / Tier 3(유료)

Challenger 4건 VALID 반박:
1. **sqlite-vec 미검증**: 실험적 프로젝트, hybrid 결합 검증 부재
2. **Groq 토큰 제한**: 6000 tok/min → 실제 분당 1.5회 (10회가 아님)
3. **3-tier 유지보수**: 3×3=9 조합 테스트 — maintainer 1명 불가능
4. **pip install 환상**: C extension + torch 의존 → 빌드 에러 이탈

### R3 — 단일 권장 경로로 수렴

Executor 최종안: **하나의 기본 구성을 완벽하게**

```
Embedding : gte-multilingual-base (768d, HuggingFace, 로컬)
LLM       : Ollama gemma2:9b-instruct
Vector DB : ChromaDB (순수 Python)
BM25      : rank_bm25 (순수 Python)
Hybrid    : RRF k=60 (순위 기반, 점수 스케일 무관)
```

Challenger 3건 반박 중 핵심:
- **BM25 정규화**: vector(0~1) vs BM25(0~∞) 스케일 차이 → **RRF로 해결** (순위 기반)
- **citation 검증 필요**: 50쿼리 실험 → 준수율 90% 미만 시 few-shot 추가
- **배포 대상 명확화**: 개발자/준개발자/비개발자 단계 분리 필요

### R4 — 수렴

Challenger가 "HF Spaces GPU 제한"과 "데이터 라이선스"를 제기했으나 자기 범위 이탈 인정. WEAK 2건.

## 확정 구성

| 구성요소 | 현재 (유료 $20~45/월) | 비용 제로 | 전환 근거 |
|----------|---------------------|----------|----------|
| Embedding | OpenAI text-embedding-3-small | gte-multilingual-base (로컬) | 한국어 특화, Apache 2.0 |
| Light LLM | gpt-4o-mini | Ollama gemma2:9b | instruction following |
| Heavy LLM | claude-sonnet | Ollama gemma2:9b | 단일화 (복잡도 제거) |
| Vector DB | Supabase pgvector | ChromaDB | pip 설치, 서버 불필요 |
| BM25 | PostgreSQL tsvector | rank_bm25 | 순수 Python |
| Hybrid | SQL RPC (weight 기반) | RRF k=60 (rank 기반) | 정규화 불필요 |

**유료 API는 옵션**: `.env`에 `OPENAI_API_KEY` 등이 있으면 자동 활성화, 없으면 로컬 사용.

## 선결 과제

1. gemma2:9b citation 준수 실험 (50쿼리)
2. ChromaDB + rank_bm25 + RRF retrieval 품질 비교 (현재 Supabase hybrid 대비)
3. GPU 감지 → 모델 자동 선택 (9b vs 2b)
