# 03. 상태머신 확장

## 문제 (R10 C10-1)

초기 상태머신(requested→processing→completed)에 실패 복구 경로 없음. processing 영구 고착.

## 최종 해소

```
requested → processing → completed
    ↑            │
    └── failed ←─┘  (재시도 3회 미만)
completed/failed → expired → (30일) → deleted
```

- failed 상태 + ErrorResult 메커니즘
- 시작 시 복구: processing + updated_at > TTL → 자동 failed
- 고착 판단: Provider TTL 기반 (500ms 지연 무시)
