# 환각 방지 (Sufficiency Gate)

> 주제2 (검색-생성) — 검색 결과가 부족할 때의 거절/제한 메커니즘

## 문제

검색 결과가 질문과 관련 없거나 불충분한데, LLM이 **자기 학습 데이터로 답변을 만들어내는** 환각 문제. RAG의 존재 이유가 "문서 기반 답변"인데, 이게 깨지면 시스템 신뢰 전체가 무너짐.

## 단독 설계에서는

프롬프트에 "문서 기반으로만 답변하라"고 지시. 그러나 **LLM이 이 지시를 따르는지 검증하는 메커니즘이 없음**. 프롬프트 지시는 "부탁"이지 "강제"가 아님.

## 적대적 검증에서 일어난 일

**R1** — Challenger: "프롬프트 지시만으로 문서 외 지식 사용이 차단된다고 전제한다. 검증 단계가 없다"

**R2** — Challenger: "검색 점수가 낮아도 답변을 생성한다면, 충분한 근거 없이 답변하는 것이다"

## 최종 설계

```
검색 결과 rerank 점수
        ↓
  score < low (0.3)  → 거절: "관련 정보를 찾을 수 없습니다"
  low ≤ score < high → 제한 답변: "관련 정보가 제한적입니다" + 경고
  score ≥ high (0.7) → 정상 답변
```

+ 후검증: 답변 내 각 주장이 인용 청크에 실제로 근거하는지 NLI 기반 확인

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 근거 부족 시 | LLM이 알아서 판단 (환각 위험) | **시스템이 점수 기반으로 차단** |
| 거절 능력 | 없음 | 2단계 임계값으로 거절/제한/정상 분기 |
| 검증 | 없음 | 후검증 (주장-근거 매칭) |
| 사용자 신뢰 | "이 답변 맞아?" 확인 불가 | confidence 표시 + 인용 제공 |
