# 스트리밍 + 인용 정합성

> 주제2 (검색-생성) — "동시에 만족 불가능"을 해소한 문장 버퍼 SSE

## 문제

사용자 경험을 위해 **답변을 실시간 스트리밍**하고 싶은데, 인용은 **문장이 완성되어야** 붙일 수 있음. 토큰 단위 스트리밍과 문장 단위 인용이 **구조적으로 충돌**.

## 단독 설계에서는

이 문제를 고려하지 않음. 스트리밍이든 인용이든 한쪽만 구현하거나, 둘 다 없음.

## 적대적 검증에서 일어난 일

**R2** — Challenger: "SSE 토큰 스트리밍과 문장 단위 인용 강제가 동일 시점에 만족 불가능하다"

이전 실험(21R)에서도 이 문제가 나왔었음:
- "미검증 내용이 먼저 스트리밍되는 순서 모순" (R8-C3, critical)
- "Stream → Format 순서로 인용 없이 스트리밍됨" (R21-C2)

## 최종 설계: 문장 버퍼 SSE

```
LLM 출력 (토큰 단위)
    ↓
문장 버퍼 (토큰 축적)
    ↓
문장 경계 감지?
    ├── Yes → 문장 + 인용 메타를 SSE 이벤트로 전송
    └── No → 계속 축적
    ↓
EOS 수신?
    └── Yes → 잔여 버퍼 flush (type: flush)
```

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 스트리밍 | 토큰 단위 (인용 불가) 또는 없음 | **문장 단위 SSE** (인용 포함) |
| 인용 시점 | 답변 완료 후 일괄 | **문장 완성 즉시** |
| 미검증 노출 | 가능 | **문장 완성 + 인용 첨부 후에만 전송** |
| EOS 처리 | 미정의 | flush 이벤트로 잔여 보장 |

## 왜 이 문제가 중요한가

스트리밍 없으면 사용자가 **수 초간 빈 화면**을 봐야 함. 인용 없으면 **답변을 신뢰할 수 없음**. 둘 다 필요한데 구조적으로 충돌 — 이걸 "문장 버퍼"라는 중간 레이어로 해소한 것이 핵심.
