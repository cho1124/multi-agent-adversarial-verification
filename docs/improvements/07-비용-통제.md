# 비용 통제

> 주제1 (청크/임베딩) — API 종속 → 로컬 우선으로 전환

## 문제

RAG 시스템은 임베딩 + 검색 + LLM 호출이 매 질의마다 발생. API에 의존하면 **사용량에 비례해 비용이 증가**하고 통제 불가.

## 단독 설계에서는

`text-embedding-3-small` (OpenAI API) 사용. 비용 계산이나 통제 전략 없음.

## 적대적 검증에서 일어난 일

**R1** — Challenger: "비용 0이라고 했으면서 Cohere Rerank API를 쓴다. 모순이다"

**R1** — Challenger: "dense 임베딩만으로 400MB라고 했는데 sparse 벡터 저장량이 누락. 실제 비용이 과소 추정됨"

이후 TCO(Total Cost of Ownership)를 전면 재작성하면서 로컬 vs API 비교가 이루어짐.

## 최종 설계

| 항목 | 로컬 | API |
|------|------|-----|
| 임베딩 | bge-m3 (GPU 추론) | text-embedding-3-large |
| Reranker | 로컬 cross-encoder | Cohere Rerank |
| 저장 | 로컬 벡터 DB | 클라우드 |
| **월 비용** | **~$15** (전기+GPU) | **~$12.50** (API 호출) |

핵심: 로컬이 약간 비싸지만 **사용량 증가에도 비용이 거의 고정**. API는 사용량에 비례하여 증가.

## 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 비용 예측 | 불가 (API 종속) | **월 고정비 산출 가능** |
| 비용 통제 | 없음 | 로컬 우선 + API 폴백 |
| 비용 모니터링 | 없음 | 질의당 비용 분해 (검색/rerank/생성/캐시) |
| 스케일 비용 | 선형 증가 | **거의 고정** (로컬) |
